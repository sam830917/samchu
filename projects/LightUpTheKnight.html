Description

In Light Up the Knight you play as the Fire Knight who has been ordered by the king of the realm to create fireworks for festivals. Push fireworks boxes to create a chain reaction and reach the final, red, fireworks box. Maneuver the fireworks boxes around non-destructible obstacles to set off fireworks boxes and destroy old crates.

Once the boxes are in place, the Fire Knight swings the Fire Sword to ignite fireworks and start the chain reaction. Then, the Knight sits back to watch the spectacle. The more boxes that are used, the better the results will be!

Features

Auto Path

  • The player can be able to tap on any square that is accessible and be able to move to that square.
  • The fire Knight should move visibly to the new square while avoiding occupied squares.

Tutorial Controller

  • In tutorial level, player can only follow the instructions.
  • Level designers can modify every step on panel which means they don't need to touch any code.

Torch System

  • Torches represent the tiers of achievement. Bronze being the lowest possible achievement and gold being the highest.
  • When the win scroll appears, it includes 3 torches that light up depending on how well you completed the level. There are 3 tiers of torches: bronze, silver and gold.
  • How much torches you earn will show in level select.

Combo System

  • Fire Knight swings the Fire Sword to ignite fireworks and start the chain reaction.
  • The chain reaction will continue to happen until all firework rockets are hit obstacles.

Auto Path

I was using A* algorithm to find a path for the player. Because in the Light Up The Knight, map is tile-base. Character is moving tile to tile, and the shortest path is not necessary. Therefore, A* algorithm is best for this game. 

public static List<Vector2Int> FindPath( GameObject[,] tilesArray, Vector2Int targetPosition, Vector2Int playerPosition )
    {
        Vector2Int mapSize = new Vector2Int(tilesArray.GetLength(0), tilesArray.GetLength(1));
        int[,] map = GetMap(tilesArray);
        var graph = new Graph(map);
        var search = new Search(graph);

        int playerIndex = TransformPositionToIndex(playerPosition, mapSize);
        int targetIndex = TransformPositionToIndex(targetPosition, mapSize);
        search.Start(graph.nodes[playerIndex], graph.nodes[targetIndex]);

        while (!search.finished)
        {
            search.Step();
        }

        List<Vector2Int> path = new List<Vector2Int>();
        //print("Search done. Path length " + search.path.Count + " iterations " + search.iterations);
        for (int i = 1;i < search.path.Count;i++)
        {
            var label = Int32.Parse(search.path[i].label);
            path.Add(TransformIndexToPosition(label, mapSize));
        }
        return path;
    }

This function return a list that contains every direction that player should go to.

Tutorial Controller

Due to in Light Up The Knight, we have 4 tutorial levels. I cannot just hard core every instruction for every level. So, I made a Tutorial Controller for level designers. I want it highly flexible and easily modify. 

[System.Serializable]
    public class Steps
    {
        public string stepName;
        public Step step;
        [Header("MOVE PLAYER")]
        public Vector2Int playerGoTo;
        [Header("MOVE BOX")]
        public Vector2Int boxFrom;
        public Vector2Int boxTo;
        [Header("TAP BOX")]
        public Vector2Int boxIndex;
    }
	
	public enum Step
    {
        MOVE_PLAYER,
        PRESS_IGNITE_BUTTON,
        MOVE_BOX,
        JUST_TAP_BOX
    };

In Steps Array, size is how many step you should follow. And Step has 4 instruction enum.

  • Move Player
  • Press Ignite Button
  • Move Specific Box
  • Tap Specific Box
Step Name just give a name for every step. For using this controller, the most important part is to select one step instruction, if level designers choose MOVE_PLAYER, and they should also set player target coords below.

When the level starts, Tutorial Controller will lock up the movement of knight, ignite button and interaction with box. And white glove will pop up to indicate the player to tap sepecific area. This controller make sure player can finish tutorial level without breaking level and misunderstanding the mechanics.
