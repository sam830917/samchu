Description

Trikaya is an atmospheric first-person puzzle game where the player explores a long-forgotten temple and pieces together a mighty and mysterious artifact to master magics to uncover the intriguing history of the extraordinary artifact.

  • Collect the Wind gem to summon powerful gusts of wind to mobilize objects.
  • Acquire the Earth gem to create a mass of rock that can be used to shoot and destroy objects, and generate great pillars that allow you to reach new heights.
  • Obtain the Fire gem to incinerate objects in your way and reveal hidden paths never traversed before.

Interactable Object

In our game, the interaction between object and spell is the game pillar. In order to maintain current objects and add new objects. We kept it highly flexible and all the object inherits the base class.

On the base classes, we used the virtual function for the general use function. In this case, a child class can easily override base function and write specific behavior.

For example, ore cube can interact with wind spell and fire spell. Therefore, ore cube should override InteractWithFire and InteractWithWind functions.

Checkpoint System

In order to save current player state, I made a check point system which can save player location, rotation and gems.

The checkpoint system contains an array of trigger notes. A trigger note includes a cube area(can be scale) when the player overlap with it, it can save current state. Every trigger note will draw in the map(only on editor mode), it is useful for the level designer to know the saving location.

Also, every trigger note includes an index. This index is for activating or deactivating another trigger node. That is to say when a trigger node got overlapped, it will deactivate the nodes that have a smaller index. This mechanism prevents when player skips the trigger nodes, and go back to the previous route which is in the wrong direction, the player will not trigger a checkpoint.

For instance, there has five trigger nodes, when player triggers 1 node, and go to 3 node, it will deactivate two 2 nodes.
